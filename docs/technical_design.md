# EcomHighlightSkill 技术设计

## 1. 设计原则

本项目不让大模型直接处理完整长视频并输出时间戳，而采用更稳定的两阶段方案：

```text
程序化生成候选片段 -> 规则/模型对候选片段打分 -> 约束选择 -> ffmpeg 剪辑导出
```

这样可以降低时间戳不稳定风险，并让评测结果可解释。

## 2. 模块划分

```text
输入校验
  -> 视频元信息读取
  -> 候选片段生成
  -> 字幕/转写对齐
  -> 电商高光评分
  -> Ark/方舟可选重排序
  -> 片段选择
  -> ffmpeg 裁剪拼接
  -> 结果报告
  -> 自动评测
```

## 3. MVP 能力

当前版本优先保证端到端闭环：

- 读取本地 mp4。
- 读取视频时长。
- 生成候选片段。
- 基于电商关键词、字幕密度、片段时长、镜头边界进行启发式评分。
- 选择高分片段并控制总时长。
- 导出 `highlight.mp4`。
- 输出 `segments.json`、`timeline_report.md`、`result.json`。
- 支持 Ground Truth 自动评测。

## 4. 后续增强方向

- 引入 PySceneDetect 提升镜头边界质量。
- 引入 Whisper 或方舟 ASR 做口播转写。
- 引入 OCR 获取画面文字。
- 抽取关键帧并调用多模态模型生成 caption。
- 让 LLM 根据关键帧描述、字幕和用户指令进行结构化评分。
- 生成 HTML 可视化评测报告。
