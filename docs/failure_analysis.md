# EcomHighlightSkill 失败案例分析模板

## 失败案例 1：选中片段画面好看但商品不可见

现象：

```text
系统选择了光线较好、背景漂亮的片段，但商品主体没有完整出现。
```

可能原因：

- 当前 MVP 缺少真正的商品主体检测。
- 字幕或指令关键词误导了片段评分。

优化结论：

- 提高商品可见度权重。
- 接入关键帧 caption 或商品检测模型。
- 对商品不可见片段进行硬过滤。

## 失败案例 2：片段边界不自然

现象：

```text
裁剪从动作中间开始，或在一句口播中间截断。
```

可能原因：

- 候选片段边界来自固定窗口。
- 镜头切分没有对齐语义边界。

优化结论：

- 结合镜头切分结果修正起止时间。
- 对片段边界前后扩展 0.5-1 秒。
- 使用字幕时间戳避免截断关键句。

## 失败案例 3：卖点重复覆盖

现象：

```text
输出 15 秒视频中多个片段都展示外观，但没有覆盖功能演示或使用场景。
```

可能原因：

- 片段选择只按分数排序。
- 缺少 `must_cover_points` 覆盖约束。

优化结论：

- 在选择阶段加入卖点去重。
- 每个必选卖点至少选择一个相关片段。
- 对重复标签片段增加惩罚。

## 失败案例 4：剪入低质量片段

现象：

```text
输出视频包含黑屏、模糊、强抖动或商品被遮挡的片段。
```

可能原因：

- 当前质量评估主要依赖文本和启发式规则。
- 缺少图像质量检测。

优化结论：

- 增加黑屏检测、模糊检测、抖动检测。
- 在 Ground Truth 中维护 `avoid_segments` 并用于扣分。
- 对低质量候选片段直接过滤。
